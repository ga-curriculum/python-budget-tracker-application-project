Students typically learn the most while building projects. During this time, you'll be self-directed to plan, build, and complete your project.

You'll need to submit some key deliverables along the way to keep you on track. To help you through this process, we've included this guide to walk you through the individual steps outlined below so that you can be sure you're not missing anything along the way.

## 1. Project info

Before you start planning, review the [Project Requirements and Guidance](../project-requirements-and-guidance/README.md) to understand what features you must implement in your project. This document also includes helpful tips for getting started.

## 2. Create a GitHub repo

Once your project proposal is approved, create the public GitHub repo that you'll use for the project. For more details, see the [Project GitHub Repo URL Deliverable](../project-github-repo-url-deliverable/README.md).

## 3. Build the project

The fun part! Start building your project! The printable [project requirements PDF](../project-requirements-and-guidance/assets/project-requirements.pdf) may be useful for checking items off as you go through the project.

## 4. Submit a URL to your completed project

Submit the URL for your the GitHub repo containing your project. For more details, see the [Project GitHub Repo URL Deliverable](../project-github-repo-url-deliverable/README.md).

## 5. Present your completed project

After you've finished the project, you'll have the opportunity to present your project to your instructors and classmates. For more details, see the [Project Presentation Requirements](../project-presentation-requirements/README.md).
